# Typing Test
#### Video Demo: https://www.youtube.com/watch?v=B8Vdy_XJEPI
This app is also deployed on render: https://typing-test-app-kp53.onrender.com

### Overview
This app is a standard typing test website. Allowing users to register an account and then take a 30 second typing test. I created this website so anyone could compare typing results against their friends, the world, or themselves. The app has an "ocean blue" theme to make the website easier on the eyes.

### Authentication & Access
When you first visit the site the user will be prompted to login via `login.html`. If they do not yet have an account they can click on the `register` link inside the navbar. This will bring them to the register form `register.html`, where they will be able to create an account. If the user attempts to register an account with a username that is already taken, or the user doesn't submit anything at all, they will be shown an apology message generated with memegen via `apology.html`. The user does not need an account to view the global leaderboard, if they wish to see the top ten they can do so freely without an account.

### Managing Your Account
 Once the user registers they will be automatically signed into their account. If the user wishes to log in to a different account they can simply logout using the navbar and then click on the login link. If the user attempts to login with an invalid login they will once again be shown an apology. If the user is unhappy with their password they may change it by clicking on the `change_password` link, which brings them to `change.html`,Every form is submitted via POST, after which Flask handles the information and updates the database `typing-test.db` accordingly. All users and test results will be stored in the database.

### Test flow
 Once a user has an account they will be redirected to the homepage where the test is explained.The user may take a 30 second typing test by either clicking the button on the homepage, or by clicking the link inside the navbar. The user will then be redirected to the test page `test.html`. The text the user will be typing over during the test will then generate 75 random but distinct words. During the test, the users input will cause the original text to be highlighted in real time, turning green if the users input matches or red if it doesn't. The test will begin only once the user begins typing into the textfield.

### Test implementation
 The words are generated by using random.sample on words.txt in python. WPM, Accuracy, and Adjusted WPM were also all calculated on the backend using python logic
#### real time highlighting
  The real time highlighting was implemented using JavaScript, by adding an input listener that begins a for loop the same length as the original text. A character comparison is run if the user has typed that far. When the comparison is triggered a check on whether the user's character matches the original text takes place. After the comparison a colored span tag is generated containing the character. If the comparison isn't run the app will simply return a white span tag containing the character. After iterating over all characters, the innerHTML of the text is replaced with the generated span tags.

 #### Timer
   The timer on input was implemented by adding an input listener to the textfield in JavaScript. Once the event is triggered an interval begins which decreases the timer every second. When the timer reaches zero the interval is cleared and the form is submitted.

  I also added some anti cheat measures that prevent the user from pasting or dropping text into the textfield, as well as preventing the the user from being able to right click and open a menu on the textfield. This was implemented using JavaScript to set up event listeners for the drop, paste, and context menu events and disabling their defaults.

---

  ### Results
 Once the user has completed a test they will be redirected to `results.html`, where their test results will be displayed. Additionally once a test has been completed by a user they will be able to visit the link `your top ten` which will display `top10.html` the users 10 best test results ordered by adjusted WPM.

### Design
#### Highlighted Text
 I originally was going to have the user type directly over the text like many other typing tests do. But ultimately decided that having the user type into a textbox that causes the original text to change color, was more unique and fun.
#### Timer
  I had also considered that instead of a timed test, I'd use a test that only submits once the user has typed the entire text. I chose to stick with the timed version as I wanted the tests to be quick, that way slow typists wouldn't have to spend up to several minutes completing a single test.
#### Flask
   I decided to use Flask because I have the most experience with it. Flask sessions also made it very simple to keep login persistence
#### sqlite3
Sqlite was used for lightweight file based database storage.

#### Werkzeug
 Werkzeug for secure password hashing inside of the database. Flask uses jinja2 which made for easy dynamic HTML rendering. Flask sessions also made it very simple to keep login persistence.

 #### Adjusted WPM
 I chose to use the Adjusted WPM metric to rank results to keep in line with modern typing tests.

### Project Structure and Files
- **static/**: Static files used to render apology, random words for the test, JavaScript to give functionality to the typing test, and CSS style
- `style.css`: custom styles to give the website a more friendly "dark mode" theme
- `test.js`: code to give the typing test (test.html) functionality chatgpt used as helper to come up with and debug code
- `words.txt`: 300 words used to generate 75 random and distinct words for the typing test
- **templates/**: HTML templates used by Flask for rendering pages
  - `apology.html`: Adapted from cs50's problem set 9 "finance" allows for memegen displays of errors
  - `change.html`: form to allow users to change their password
  - `index.html`: Homepage
  - `layout.html`: A simple layout using jinja to allow for both a nice website appearance, nav bar and for  extending other html pages
   - `leaderboard.html`: Display top scores currently held in the database sorted by adjusted_wpm
  - `login.html`: A simple login form for users
  - `register.html`: A  simple register form for users to create a new account
  - `results.html`: Displays the results of the test a user has just taken.
  - `test.html`: The interface of the typing test with a box form to type into and a 30 second timer
  - `top10.html`: Displays the user's 10 best results out of every test they've taken
  - **app.py**: Main Flask application file containing routes, session management, and database interactions.
  - **helpers.py**: Contains helper functions like `apology`, `login_required`, `get_db`, and `close_db`. both functions apology and login_required are from cs50 problem set 9 "finance"
  - `README.md`: The file you're currently reading!
  - `requirements.txt`: The required installs for this app to work properly.
- **typing-test.db**: SQLite database with `users` and `results` tables.
